# 🤖 คลังระบบเทรดอัตโนมัติอัจฉริยะ (BotTrade Workspace)

ยินดีต้อนรับสู่พื้นที่ทำงานคลังระบบเทรดอัตโนมัติ (Automated Trading Workspace) สำหรับแพลตฟอร์ม MetaTrader 5 (MT5) โดยเน้นการพัฒนาและเพิ่มประสิทธิภาพการทำกำไรบนราคาทองคำ (XAUUSD / GOLD) ในไทม์เฟรม H4 

โปรเจกต์นี้เป็นการรวบรวมระบบเทรดปัญญาประดิษฐ์ (Machine Learning) และระบบคำนวณทางคณิตศาสตร์ขั้นสูง 2 รูปแบบ ซึ่งถูกพัฒนาและทดสอบย้อนหลัง (Backtesting) ด้วยข้อมูลราคาระดับความแม่นยำสูง (1-Minute OHLC) เพื่อให้ได้ระบบการลงทุนที่มีความเสถียรและยืดหยุ่นภายใต้สภาวะตลาดจริง

## 1. ระบบเทรดอัตโนมัติทั้ง 2 รูปแบบ (The Core Trading Systems)

ระบบเทรดในโปรเจกต์นี้ถูกแยกออกตามลักษณะสถาปัตยกรรมโครงสร้างและการคำนวณทางคณิตศาสตร์ เพื่อตอบสนองต่อสภาวะตลาดทองคำที่แตกต่างกันออกไป โดยมีรายละเอียดเบื้องต้นดังนี้:

### 1.1 Machine Learning: Lorentzian Classification (บอทเทรดรูปแบบ k-NN ในมิติ Lorentzian) 🌌

ระบบนี้ใช้หลักการการพยากรณ์เชิงการเรียนรู้ของเครื่อง (Machine Learning) ประเภท **k-Nearest Neighbors (k-NN)** ซึ่งเป็นการค้นหาชุดข้อมูลที่มีคุณลักษณะ (Features) ในอดีตที่ใกล้เคียงกับราคาปัจจุบันมากที่สุดเพื่อทำนายทิศทางแนวโน้มราคาในอนาคต

*   **ทฤษฎีและการคำนวณทางคณิตศาสตร์:**
    *   **Lorentzian Distance:** ในระบบวิเคราะห์ทั่วไปมักใช้ระยะทางแบบยุคลิด (Euclidean Distance) ซึ่งเป็นการวัดระยะทางตรง แต่มีข้อเสียสำคัญคือจะอ่อนไหวอย่างมากต่อข้อมูลที่ผิดปกติ (Outliers) เช่น แท่งเทียนที่กระชากตัวรุนแรงในช่วงข่าวนอนฟาร์ม หรือข่าวดอกเบี้ย FOMC ระบบนี้จึงเปลี่ยนมาใช้ **Lorentzian Distance** ซึ่งใช้ฟังก์ชันลอการิทึม $\ln(1 + |x_i - y_i|)$ ครอบการวัดระยะทาง ส่งผลให้ช่วยลดและควบคุมน้ำหนักของข้อมูลผิดปกติ (Outliers Neutralization) ทำให้ระบบมีความเสถียร ไม่ส่งสัญญาณเทรดผิดพลาดในสภาวะราคากระชากรุนแรง
    *   **5-Dimensional Feature Engineering:** ดึงค่าทางเทคนิค 5 ตัวแปรหลัก ได้แก่ RSI, WaveTrend, CCI และ ADX มาทำการย่อสเกลด้วยเกณฑ์ Min-Max Normalization ให้อยู่ในช่วง $0.0$ ถึง $1.0$ เสมอ เพื่อให้แต่ละตัวแปรมีอิทธิพลในการคำนวณระยะทางที่สมดุลกัน
*   **จุดเด่นทางเทคนิค:** มีการใช้ตัวกรองเพื่อนบ้านระดับคุณภาพสูง (75th Percentile Rule) การเว้นระยะเวลาเพื่อป้องกันข้อมูลซ้ำซ้อน (Chronological Modulo Spacing) และการเพิ่มความเร็วการคำนวณสูงสุดถึง **10 เท่า (Static Weights Table Lookup)** ช่วยให้การทดสอบและทำงานจริงบนโปรแกรม MT5 ทำได้อย่างมีประสิทธิภาพสูงสุด
*   **รายละเอียดฉบับสมบูรณ์:** สามารถศึกษารายละเอียดสมการคณิตศาสตร์ ผลลัพธ์การทดสอบ และขั้นตอนการติดตั้งได้ที่คู่มือหลัก 👉 **[README.md ของ Lorentzian Classification](./Machine%20Learning%20Lorentzian%20Classification/README.md)**

### 1.2 Uptrick ML Kernel Regression (บอทเทรดรูปแบบ Nadaraya-Watson Kernel Regression) 📈

ระบบเทรดตามแนวโน้มที่มีประสิทธิภาพสูง โดยประยุกต์ใช้เทคนิคประมาณค่าการถดถอยที่ไม่พึ่งพารามิเตอร์ (Non-parametric Regression) ด้วยเครื่องมือ **Nadaraya-Watson Gaussian Kernel Estimator** 

*   **ทฤษฎีและการคำนวณทางคณิตศาสตร์:**
    *   **Gaussian Kernel Regression:** ระบบจะทำหน้าที่สร้างเส้นประมาณการราคาระดับเรียบเนียน (Smoothed Trend Line) ปราศจากสัญญาณรบกวน (Noise Filtered) โดยให้น้ำหนักแท่งเทียนที่อยู่ใกล้เคียงผ่านฟังก์ชันคาร์เนลแบบเกาส์เซียน (Gaussian Kernel)
    *   **Volatility-Scaled Adaptive Bandwidth:** เพื่อแก้ไขข้อจำกัดของเส้นค่าเฉลี่ยทั่วไปที่มีความล้าหลังสูง (Lagging) ระบบนี้ได้ประยุกต์ใช้ **Adaptive Bandwidth** โดยปรับขนาดแถบความกว้างน้ำหนักตามดัชนีความผันผวนของราคาจริง (ATR) ส่งผลให้ในสภาวะตลาดเป็นแนวโน้มรุนแรง (Breakout) แถบความกว้างจะหดตัวลงโดยอัตโนมัติ ทำให้เส้นแนวโน้มตอบสนองตามราคาได้อย่างรวดเร็ว และในสภาวะตลาดเป็นไซด์เวย์ แถบความกว้างจะขยายออกเพื่อกรองสัญญาณหลอก (Whipsaw Filter)
*   **จุดเด่นทางเทคนิค:** ระบบได้รับการพัฒนาให้เป็น **Non-Repainting Engine 100%** ในภาษา MQL5 สัญญาณซื้อขายจะทำงานเฉพาะเมื่อแท่งเทียนปิดตัวลงอย่างสมบูรณ์แล้วเท่านั้น ทำให้ผลการรันจริงตรงตามการจำลองย้อนหลังทุกประการ
*   **รายละเอียดฉบับสมบูรณ์:** สามารถศึกษาการตั้งค่าพารามิเตอร์ สูตรการคำนวณความชัน และสถิติเพิ่มเติมได้ที่คู่มือหลัก 👉 **[README.md ของ Uptrick ML Kernel Regression](./Uptrick%20ML%20Kernel%20Regression/README.md)**

## 2. ระบบสถาปัตยกรรมร่วมของแกนกลางการเทรด (Shared Core Architecture)

บอทเทรดทั้ง 2 ตัว ได้รับการพัฒนาขึ้นบนระบบจัดการคำสั่งเทรดแกนกลางประสิทธิภาพสูง เพื่อความมั่นคงปลอดภัยของเงินทุนและการทำงานอย่างต่อเนื่อง โดยมีฟังก์ชันเด่นร่วมกันดังนี้:

### 1️⃣ Virtual / Stealth Stop Loss & Take Profit (การควบคุมออเดอร์แบบซ่อนเร้น) 🥷
*   **จุดประสงค์:** โบรกเกอร์บางรายอาจดักจับทิศทางราคาที่มีการตั้งจุดตัดขาดทุน (SL) ในระบบของเซิร์ฟเวอร์หลักเพื่อดึงราคามาสะกิดปิดออเดอร์ก่อนกลับตัว (Stop Hunting)
*   **การทำงาน:** บอทจะทำการส่งคำสั่งซื้อขายไปยังเซิร์ฟเวอร์โดยกำหนดค่า **SL = 0 และ TP = 0** เสมอ (ไม่มีระดับราคาแสดงบนหน้าจอระบบของโบรกเกอร์) โดยที่ระดับ SL/TP จริงจะถูกซ่อนบันทึกไว้ภายในหน่วยความจำถาวรของโปรแกรมที่เรียกว่า **MetaTrader 5 Global Variables (GV)**
*   **ความเสถียร:** Global Variables จะคงทนอยู่บนฮาร์ดไดรฟ์เครื่องคอมพิวเตอร์ของคุณ แม้เกิดเหตุการณ์ระบบโปรแกรมถูกปิดกะทันหัน เครื่องดับ หรือไฟตก เมื่อโปรแกรมกลับมาทำงาน บอทจะดึงค่าเป้าหมายเดิมมาประมวลผลต่อได้ทันที และส่งคำสั่งปิดออเดอร์ตรงสู่ตลาดทันทีเมื่อราคาปัจจุบันสัมผัสเป้าหมาย

### 2️⃣ 1:1 Breakeven Protection (การเลื่อนระดับราคาป้องกันหน้าทุน) 🛡️
*   เพื่อล็อกความเสี่ยงในการสูญเสียเงินทุนให้เป็นศูนย์อย่างรวดเร็ว (Risk-Free Trade) เมื่อระบบเข้าออเดอร์และราคาวิ่งไปในทิศทางที่ถูกต้องจนถึงระยะสัดส่วน **1:1** ของความเสี่ยง (ครึ่งทางของเป้าหมาย Take Profit)
*   บอทจะทำการเลื่อนระดับ Virtual Stop Loss มาตั้งป้องกันไว้ที่ระดับ **ราคาเข้าออเดอร์ (Entry Price) + 10 จุด** เพื่อครอบคลุมค่าสเปรดและค่าธรรมเนียมคอมมิชชันโดยอัตโนมัติ

### 3️⃣ Asynchronous LINE OA Alert (ระบบแจ้งเตือนแบบแยกเธรด) 💬
*   การยิงคำขอส่งข้อมูลผ่านเว็บ (WebRequest) ไปยังระบบแจ้งเตือนภายนอก มักทำให้เธรดการประมวลผลกราฟหลักของโปรแกรม MT5 ค้างลงชั่วคราวเป็นเวลา 0.5 ถึง 2 วินาที ซึ่งอาจส่งผลเสียต่อการทำงานของคำสั่งซื้อขายจริง
*   ระบบจึงแยกการทำงานออกอย่างอิสระ โดยบอทหลักจะทำหน้าที่เพียงส่งสัญญาณเตือนผ่าน `EventChartCustom` (คำสั่งเหตุการณ์ภายในชาร์ต) และมีบอทผู้ช่วยเบื้องหลัง **`Uptrick_LINE_Helper.mq5`** คอยดึงอีเวนต์ดังกล่าวออกมารวบรวมและยิงข้อมูลผ่าน LINE Messaging API ให้แบบคู่ขนาน (Asynchronous) ทำให้หน้าจอและระบบการยิงออเดอร์หลักมีความลื่นไหลเต็มร้อยเปอร์เซ็นต์

## 5. บทวิเคราะห์ปัจจัยความเสี่ยงในการเทรดจริงและการควบคุม (Live Trading Risks & Mitigation)

การเปลี่ยนผ่านจากการทดสอบย้อนหลัง (Backtesting) ไปสู่การรันระบบเทรดจริงบนตลาดบัญชีจริง (Live CFD Trading) นั้นมีปัจจัยความเสี่ยงภายนอกระบบหลายประการ ซึ่งนักลงทุนจำเป็นต้องตระหนักและประเมินระบบเพื่อเตรียมมาตรการป้องกันเชิงรุก (Proactive Risk Mitigation):

### 5.1 สถิติจำนวนผู้รอดชีวิตในตลาด CFD (CFD Success Rates Statistics)
ตามข้อมูลข้อบังคับและสถิติจากหน่วยงานกำกับดูแลการเงินสากล (FCA, ESMA, ASIC) พบว่ามีบัญชีเทรดรายย่อยเฉลี่ยถึง **74% ถึง 89% ที่ประสบผลขาดทุน** ในตลาด CFD โดยมีเพียงประมาณ **1% ถึง 5%** เท่านั้นที่สามารถสร้างผลกำไรสุทธิได้อย่างยั่งยืนในระยะยาว (มากกว่า 1 ปีขึ้นไป) เนื่องจากผลกระทบของค่าธรรมเนียมแฝง (Transaction Costs), การใช้เลเวอเรจสูงเกินขอบเขตเงินทุน (Over-Leveraging), และปัญหาทางอารมณ์ในการตัดสินใจลงทุน

### 5.2 ปัจจัยความเสี่ยงเชิงระบบเทรดที่ยังไม่ได้ปิด (Unmitigated Risks)
ในการนำ EAs ทั้งสองระบบไปรันบนตลาดจริง มีปัจจัยความเสี่ยงเชิงกายภาพ 3 ประการหลักที่ไม่ได้ถูกแสดงในระบบจำลองสถิติย้อนหลัง:

1.  **ความเสี่ยงด้านเสถียรภาพฝั่งเซิร์ฟเวอร์และการเชื่อมต่อ (Server & Connection Latency Risk):**
    *   **ปัญหา:** ระบบใช้การควบคุมคำสั่งปิดแบบซ่อนเร้น (Stealth SL/TP) ซึ่งคำสั่งที่ส่งไปฝั่งโบรกเกอร์จะมีค่า `SL = 0` และ `TP = 0` เสมอ โดยบันทึกระดับเป้าหมายจริงไว้บนหน่วยความจำถาวรภายในของ MT5 (Global Variables)
    *   **ความเสี่ยง:** หากเซิร์ฟเวอร์ VPS เกิดระบบล่ม, เครื่องคอมพิวเตอร์ดับ, สัญญาณอินเทอร์เน็ตขาดหาย หรือระบบโปรแกรม MT5 ค้างกะทันหัน ระดับการป้องกันตัดขาดทุนของพอร์ตจะสูญหายไปทันที หากเกิดเหตุการณ์ราคาเคลื่อนไหวสวนทางอย่างรุนแรงพอร์ตอาจเกิดผลเสียหายสูงสุดได้
2.  **ความเสี่ยงด้านสเปรดถ่างและการคลาดเคลื่อนราคา (Spread Widening & Slippage Risk):**
    *   **ปัญหา:** ในช่วงเวลาที่มีความผันผวนสูงมาก เช่น การประกาศข่าวเศรษฐกิจสำคัญ (NFP, CPI, FOMC) สเปรดการซื้อขายจะถ่างตัวออกกว้างกว่าปกติหลายเท่าตัว
    *   **ความเสี่ยง:** คำสั่งปิดออเดอร์เสมือน (Market Close Order) ที่ถูกส่งออกไปในเสี้ยววินาทีดังกล่าวอาจประสบปัญหาการคลาดเคลื่อนของราคา (Slippage) ทำให้อัตราสัดส่วนผลตอบแทนต่อความเสียหายแย่กว่าสถิติการจำลองย้อนหลัง
3.  **ความเสี่ยงจากการเปลี่ยนแปลงพฤติกรรมตลาด (Market Regime Shift Risk):**
    *   **ปัญหา:** โหมดการทำงาน Buy Only ที่ให้ผลลัพธ์เติบโตอย่างมหาศาลนั้น คำนวณภายใต้สภาวะตลาดทองคำที่เป็นเทรนด์ขาขึ้นเชิงโครงสร้างที่แข็งแกร่ง (Strong Bullish Regime)
    *   **ความเสี่ยง:** หากแนวโน้มใหญ่ของราคาทองคำเกิดการเปลี่ยนโครงสร้างเป็นตลาดหมีระยะยาว (Bear Market) หรือสภาวะบีบตัวออกด้านข้างที่ไม่มีทิศทาง (Choppy Sideways) บอทจะยังคงเปิดคำสั่งช้อนซื้อ (Buy the Dip) สวนทางกับราคาที่ปรับตัวลงไปเรื่อยๆ จนส่งผลให้เกิดการตัดขาดทุนต่อเนื่อง (Drawdown Loop)

### 5.3 มาตรการและแนวทางปฏิบัติในการปิดความเสี่ยงเชิงโครงสร้าง (Proactive Measures)
เพื่อปิดรอยต่อระหว่างระบบจำลองและความจริง แนะนำให้พิจารณาใช้มาตรการทางเทคนิคดังนี้:

*   **การกำหนดระดับตัดขาดทุนป้องกันภัยฉุกเฉิน (Emergency Hard Stop Loss):**
    *   ปรับปรุงค่าการส่งคำสั่งโดยกำหนดให้มี Hard SL ส่งไปยังเซิร์ฟเวอร์โบรกเกอร์ด้วย โดยตั้งไว้ที่ระดับราคาที่ห่างไกลกว่า Virtual SL เล็กน้อย (เช่น ตั้ง Virtual SL ไว้ที่ 1,500 จุด และมี Hard SL เผื่อไว้ที่เซิร์ฟเวอร์ที่ 2,000 จุด) เพื่อเป็นหลักประกันความเสถียรในกรณีระบบเชื่อมต่อขาดหาย
*   **การเลือกใช้ระบบเครือข่ายความหน่วงต่ำ (Low Latency VPS Selection):**
    *   คัดเลือกและเช่าซื้อบริการ VPS ที่มีที่ตั้งอยู่ในศูนย์ข้อมูลเดียวหรือใกล้เคียงกับที่ตั้งเซิร์ฟเวอร์ของโบรกเกอร์ (Latency < 5 ms) เพื่อป้องกัน Slippage ในจังหวะยิงปิดคำสั่งแบบตลาด
*   **การเปิดตัวกรองแนวโน้มใหญ่ข้ามไทม์เฟรม (Regime Filter Activation):**
    *   เปิดใช้งาน EMA 200 Filter ใน EA เพื่อสั่งให้ระบบหยุดทำการเปิดสัญญาณหน้า Buy โดยอัตโนมัติหากราคาปิดยืนต่ำกว่าระดับ EMA 200 ในไทม์เฟรมใหญ่ (เช่น H4 หรือ D1) ป้องกันความเสียหายในกรณีตลาดปรับเปลี่ยนโครงสร้างหลักเป็นเทรนด์ขาลง
*   **การทำ Forward Test บนบัญชีทดลอง (Demo Testing):**
    *   รันระบบทดสอบบนบัญชีจริงเสมือน (Demo Account) อย่างน้อย 1 ถึง 3 เดือนเพื่อสังเกตการจับคู่ราคาสเปรด การตอบสนองของระบบแจ้งเตือน และเสถียรภาพการประมวลผลของ VPS ก่อนย้ายเงินทุนจริงเข้าสู่ระบบ

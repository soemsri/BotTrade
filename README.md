## 1. ทฤษฎีคณิตศาสตร์และการคำนวณเบื้องหลัง (Mathematical Foundation)

อินดิเคเตอร์ทำงานโดยใช้ **Nadaraya-Watson Kernel Estimator** ซึ่งเป็นเทคนิคการเรียนรู้ของเครื่องแบบไม่อาศัยพารามิเตอร์ (Non-parametric Machine Learning) เพื่อหาแนวโน้มราคาที่ปราศจากสัญญาณรบกวน (Noise)

### 1.1 Nadaraya-Watson Gaussian Kernel
สูตรคำนวณหลักในการประเมินค่าความถดถอย (Regression) ณ แท่งเทียนปัจจุบัน $x$ จากข้อมูลในอดีต $N$ แท่ง:

$$y(x) = \frac{\sum_{i=1}^{N} K\left(\frac{x - x_i}{h}\right) \cdot Price_i}{\sum_{i=1}^{N} K\left(\frac{x - x_i}{h}\right)}$$

โดยที่ฟังก์ชันน้ำหนักตัวกรอง **Gaussian Kernel $K(u)$** คำนวณจาก:

$$K(u) = \exp\left(-\frac{u^2}{2}\right)$$

*   $h$ คือ **Bandwidth (แถบความกว้างน้ำหนัก)** ยิ่งมีค่าสูง เส้นถดถอยจะยิ่งเรียบ แต่จะแลกมาด้วยความล้าหลัง (Lag) ที่เพิ่มขึ้น

### 1.2 Volatility-Scaled Adaptive Bandwidth (การปรับความกว้างแบนด์ตามความผันผวน)
เพื่อแก้ปัญหาเส้นตอบสนองช้าเกินไปในสภาวะตลาดเบรคเอาท์รุนแรง (Breakout) ระบบได้ติดตั้งสูตร **Adaptive Bandwidth** โดยนำตัวบ่งชี้ **ATR (Average True Range)** มาปรับแต่งค่า $h$ แบบไดนามิก:
1. หาระยะห่างของราคากับเส้นประมาณการ (Residuals)
2. นำส่วนเบี่ยงเบนมาตรฐานของ Residuals มาปรับสเกลร่วมกับ ATR
3. เมื่อตลาดมีความผันผวนสูง (ราคาวิ่งแรง) ค่า $h$ จะถูกบีบให้แคบลงโดยอัตโนมัติ ทำให้เส้นตอบสนองเข้าหาปัจจุบันอย่างรวดเร็ว (ลดความหน่วงล้าหลัง)
4. เมื่อตลาดเป็นไซด์เวย์ (ราคานิ่ง) ค่า $h$ จะกว้างออกเพื่อกรองสัญญาณหลอก

### 1.3 ปราศจากการวาดซ้ำย้อนหลัง (Non-Repainting Engine)
เนื่องจากฟังก์ชัน NW Kernel บน TradingView มักมีปัญหาการ Repaint (คำนวณย้อนกลับทำให้อดีตเปลี่ยน) ทางทีมพัฒนาได้แก้ไขปัญหานี้ในฝั่ง MQL5 โดยกำหนดเงื่อนไขให้ **สัญญาณ BUY/SELL ทำงานเฉพาะเมื่อ "แท่งเทียนนั้นๆ ปิดลงสมบูรณ์แล้วเท่านั้น" (Completed Bar Close)** โดยใช้ดัชนีแท่งเทียน $i=1$ แทน $i=0$ ทำให้หน้าเทรดย้อนหลังที่เห็นในการเทสต์สอดคล้องกับการเทรดจริงบนกราฟ Real-time 100%

## 2. สถาปัตยกรรมและการทำงานของ Trading EA (EA Architecture)

บอทเทรดถูกออกแบบขึ้นภายใต้หลักความปลอดภัยสูงสุดในการควบคุมเงินทุน และการพรางพอร์ตจากการถูกดักล่าราคา (Stop Hunting)

### 2.1 Stealth / Virtual Stop Loss (SL/TP ซ่อนเร้น)
*   **ความท้าทาย:** โบรกเกอร์บางรายอาจดักเห็นราคา Stop Loss ในระบบและทำการลากไส้เทียนมากวาดออเดอร์แล้วกลับทาง (Stop Hunting)
*   **ทางแก้:** บอทยิงออเดอร์ไปยังเซิร์ฟเวอร์โบรกเกอร์โดยตั้งค่า **SL = 0 และ TP = 0** เสมอ
*   **การควบคุม:** ระบบจะบันทึกระดับราคาตัดขาดทุน (SL) และทำกำไร (TP) จริงไว้บน **MetaTrader 5 Global Variables** ซึ่งเป็นหน่วยความจำถาวรภายในเครื่องคอมพิวเตอร์ของคุณ
*   **ความเสถียร:** Global Variables มีความคงทนสูง หากโปรแกรม MT5 ถูกปิดกะทันหัน, เครื่องคอมพิวเตอร์ดับ หรือมีการสลับไทม์เฟรมกราฟ ตัวแปรราคานี้จะไม่สูญหาย บอทจะโหลดราคากลับมารันต่อได้อย่างไร้รอยต่อ และทำการยิงคำสั่งปิดออเดอร์ตรงเข้าเซิร์ฟเวอร์ทันทีเมื่อราคาปัจจุบันสัมผัสผ่านเป้าหมาย

### 2.2 1:1 Breakeven Protection (การเลื่อนระดับป้องกันหน้าทุน)
เพื่อล็อกผลกำไรและลดความเสี่ยงให้เหลือ $0 (Free Trade) อย่างรวดเร็ว:
*   เมื่อคุณตั้งอัตราส่วนความเสี่ยงต่อผลตอบแทนสูงสุด (เช่น RR 1:2)
*   ทันทีที่ทิศทางราคาเคลื่อนที่บวกไปจนถึงอัตราส่วน **1:1** (ครึ่งทางของเป้าหมาย TP เช่น บวก 15.00 USD สำหรับขนาดระยะ SL 15.00 USD)
*   บอทจะเลื่อนเส้น Virtual SL มาดักไว้ที่ **จุดราคาเข้าออเดอร์ (Entry Price) + 10 จุด (เพื่อครอบคลุมค่าสเปรดและค่าคอมมิชชัน)**

### 2.3 Trend Filter (EMA 50) & Time Filter (ช่วงเวลาที่มีปริมาณการซื้อขายสูง)
*   **EMA 50 Filter:** ป้องกันสัญญาณเทรดสวนแนวโน้มใหญ่ บอทจะอนุมัติออเดอร์ฝั่ง **BUY เท่านั้นเมื่อราคาปิดอยู่เหนือเส้น EMA 50** และอนุมัติออเดอร์ฝั่ง **SELL เท่านั้นเมื่อราคาปิดอยู่ใต้เส้น EMA 50** บนกราฟ H4
*   **Session Time Filter:** กรองช่วงเวลาเงียบไร้ปริมาณซื้อขาย (เช่น ตลาดเอเชียตอนเช้า ซึ่งมักเกิดสัญญาณหลอก Whipsaw) โดยบอทจะเริ่มเปิดสัญญาเฉพาะช่วงเวลา **08:00 ถึง 18:00 (เวลาเซิร์ฟเวอร์)** ซึ่งตรงกับช่วงเวลา **13:00 ถึง 23:00 (เวลาไทย)** ซึ่งเป็นช่วงเวลาทับซ้อนของตลาดยุโรปและอเมริกาที่มีแนวโน้มเบรคเอาท์รุนแรงจริง

## 3. ตารางเปรียบเทียบผลลัพธ์การทดสอบย้อนหลัง (Backtest Performance Comparison)

สถิติการรดสอบย้อนหลัง (High-Fidelity Backtesting ระดับ 1-Minute OHLC) บนคู่เงิน **GOLD (XAUUSD / `GOLDm#`)** ไทม์เฟรม **H4** เงินทุนตั้งต้น **1,000 USD** ระยะเวลา **01/01/2025 ถึง 25/05/2026**

| ตัววัดผลความเสถียร (Metrics) | บัญชีขนาด Lot 0.1 (สเกลเริ่มต้น) | บัญชีขนาด Lot 1.0 (สเกลระดับสูง) | บัญชีขนาด Lot 10.0 (เลเวอเรจสูงเป็นพิเศษ) 🔥 |
| :--- | :---: | :---: | :---: |
| **กำไรสุทธิสะสม (Net Profit)** | **+19.22 USD** | **+192.70 USD (+19.27%)** | **+1,927.00 USD (+192.70%)** |
| **ค่าสัดส่วนกำไร (Profit Factor)** | **1.17** | **1.17** | **1.17** |
| **ผลตอบแทนคาดหวังต่อครั้ง (Expected Payoff)** | **+0.13 USD** | **+1.27 USD** | **+12.68 USD** |
| **ยอดขาดทุนสะสมสูงสุด (Maximal Drawdown)** | **16.86 USD (1.65%)** | **168.45 USD (13.80%)** | **1,684.50 USD (52.47%)** |
| **จำนวนครั้งที่เทรดทั้งหมด (Total Trades)** | **152 ครั้ง** | **152 ครั้ง** | **152 ครั้ง** |
| **อัตราการชนะฝั่งขาย (Short Trades %)** | **32.65%** | **32.65%** | **32.65%** |
| **อัตราการชนะฝั่งซื้อ (Long Trades %)** | **30.10%** | **29.13%** | **29.13%** |

### ⚠️ คำชี้แจงด้านความเสี่ยงกรณี Lot 10.0 (ไขข้อสงสัยพอร์ตล้าง)
จากการรันผลเทสต์ย้อนหลัง แม้ระบบจะดึงยอดความสูญเสียสูงสุดถึง **1,684.50 USD** ซึ่งเกินทุนตั้งต้น 1,000 USD แต่ **พอร์ตไม่ถูกล้าง (ไม่โดนบังคับปิดพอร์ต)** 

*   **วิเคราะห์สาเหตุ:** หน้าเทรดในช่วงเริ่มแรกสร้างกำไรสะสมอย่างสม่ำเสมอจนพอร์ตเติบโตขึ้นไปถึงประมาณ **3,200+ USD** จากนั้นสภาวะ Drawdown ครั้งใหญ่ที่สุดจึงเกิดขึ้น ส่งผลให้ Equity ที่สูง ณ ขณะนั้นทำหน้าที่เป็นเบาะรองรับแรงกระแทกได้เป็นอย่างดี
*   **คำเตือนความเสี่ยงจริง:** หากคุณนำกลยุทธ์ Lot 10.0 ไปรันบนบัญชีจริง แล้วตลาดเกิดผันผวนตกอยู่ในช่วง Drawdown ตั้งแต่วันแรกๆ ก่อนที่จะสะสมกำไร **พอร์ตของคุณจะล้างทันที** ดังนั้นเพื่อสัดส่วนการบริหารเงินทุนที่ยั่งยืน (Capital Preservation) แนะนำการกำหนดขอบเขตความเสี่ยงไว้ที่ระดับ **Lot 0.1 หรือ Lot 1.0 เป็นหลัก**

## 4. ระบบส่งสัญญาณเตือนผ่าน LINE OA (LINE Messaging API)

เนื่องจากระบบ **LINE Notify ยุติการให้บริการทั่วโลก** โปรเจกต์นี้จึงใช้สถาปัตยกรรม **LINE Messaging API** ในการส่งสัญญาณเตือนอย่างเป็นทางการ

### 4.1 สถาปัตยกรรมแบบ Asynchronous (ไม่บล็อกชาร์ต)
หากบอทยิง WebRequest หา LINE โดยตรงบนเธรดหลัก (Main Chart Thread) ตัวชาร์ตและคำสั่งเทรดจะหยุดนิ่งชั่วคราว (Freeze) นาน 0.5-2 วินาทีในระหว่างรอนอนุมัติจากเซิร์ฟเวอร์ LINE ซึ่งถือเป็นเรื่องอันตรายต่อการส่งคำสั่งตัดขาดทุน 

เราแก้ปัญหานี้โดย:
1. ตัวอินดิเคเตอร์/EA หลัก ทำหน้าที่ตรวจจับเงื่อนไขและยิงสัญญาณผ่านฟังก์ชัน **`EventChartCustom` (ส่งเหตุการณ์เบื้องหลัง)**
2. บอทผู้ช่วย **`Uptrick_LINE_Helper.mq5`** (รันเบื้องหลังอีกตัว) ทำหน้าที่คอยดึงอีเวนต์นี้ออกมารวบรวมและยิงส่ง `WebRequest` หา LINE แบบ Asynchronous (แยกเธรดการทำงาน) ทำให้ระบบเทรดและกราฟชาร์ตหลักลื่นไหลตลอดเวลา

## 5. ขั้นตอนการติดตั้งและใช้งานทีละขั้นตอน (Installation Guide)

### 📌 ขั้นตอนที่ 5.1: อนุญาตระบบเชื่อมต่อนอกโปรแกรมใน MT5
1.  เปิดโปรแกรม **MetaTrader 5**
2.  ไปที่เมนู **Tools (เครื่องมือ) > Options (ตัวเลือก)** หรือกดปุ่ม `Ctrl + O`
3.  คลิกที่แท็บ **Expert Advisors (ที่ปรึกษาผู้เชี่ยวชาญ)**
4.  ติ๊กเลือกเครื่องหมายถูกที่ช่อง **"Allow WebRequest for listed URL"** (อนุญาตคำขอเว็บบน URL ที่ระบุ)
5.  ดับเบิ้ลคลิกเพิ่ม URL นี้: `https://api.line.me`
6.  คลิก **OK**

### 📌 ขั้นตอนที่ 5.2: คัดลอกไฟล์ไปยังโปรแกรม MT5
1.  ในโปรแกรม MT5 ไปที่เมนู **File (ไฟล์) > Open Data Folder (เปิดโฟลเดอร์ข้อมูล)**
2.  เปิดเข้าไปยังโฟลเดอร์ `MQL5`
3.  นำไฟล์ไปจัดวางตามพาธดังนี้:
    *   คัดลอกไฟล์ `Uptrick_ML_Kernel_Regression.ex5` และ `Uptrick_LINE_Helper.ex5` ไปวางไว้ที่โฟลเดอร์ `MQL5/Indicators/`
    *   คัดลอกไฟล์ `Uptrick_ML_Kernel_Trading_EA.ex5` ไปวางไว้ที่โฟลเดอร์ `MQL5/Experts/`
4.  กลับมาที่พาเนล **Navigator (เนวิเกเตอร์)** ในหน้าจอ MT5 คลิกขวาที่หัวข้อ Indicators หรือ Expert Advisors แล้วเลือก **Refresh (รีเฟรช)**

### 📌 ขั้นตอนที่ 5.3: เริ่มรันบอทเทรดและระบุการตั้งค่า
1.  เปิดกราฟทองคำ (**XAUUSD** หรือบัญชีไมโคร **GOLDm#**) เลือกไทม์เฟรม **H4**
2.  ลากบอท **`Uptrick_ML_Kernel_Trading_EA`** จากเมนูเนวิเกเตอร์มาปล่อยลงบนกราฟ H4
3.  หน้าต่างการตั้งค่าจะปรากฏขึ้น ให้ไปที่แท็บ **Common (ทั่วไป)** ติ๊กถูกที่ช่อง **"Allow Algo Trading"**
4.  สลับมายังแท็บ **Inputs (พารามิเตอร์เข้า)** แล้วกำหนดพารามิเตอร์ดังนี้:
    *   `InpLotSize`: ขนาดความเสี่ยง (แนะนำ 0.1 หรือ 1.0 เพื่อความปลอดภัยสูงสุด)
    *   `InpStrategyMode`: เลือกโหมดกลยุทธ์เป็น `STRAT_BREAKOUT` (แนะนำสำหรับทองคำ H4)
    *   `InpUseEmaFilter`: ตั้งเป็น `true`
    *   `InpEnableBreakeven`: ตั้งเป็น `true` (เพื่อดึง SL ล็อกหน้าทุน)
    *   `InpEnableTrailing`: ตั้งเป็น `false` (ปิดเพื่อให้เป้าหมายชน TP 1:2 เต็มเปี่ยม)
    *   `InpSlMode`: `SL_MODE_FIXED`
    *   `InpFixedSlPoints`: `1500` (15.00 USD / ระยะตัดขาดทุน 1500 จุด)
    *   `InpRiskRewardRatio`: `2.0` (ระยะทำกำไร 3000 จุด)
5.  กด **OK**
6.  ตรวจสอบให้มั่นใจว่าปุ่ม **Algo Trading (การซื้อขายอัตโนมัติ)** บริเวณเมนูด้านบนของโปรแกรม MT5 แสดงไฟสีเขียวสว่างอยู่!
